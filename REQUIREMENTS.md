# 여행 앱 기능 개선 요구사항

> 기준 날짜: 2026-04-10

---

## 1. 일정 수정 기능

**현황:** 장소 추가/삭제만 가능. 방문 시간·이름·주소 수정 불가.

**요구사항:**
- 장소 행의 텍스트 영역 탭 → 수정 모달 열기
- 수정 가능 필드: 방문 시간(visit_time), 장소 이름(name)
- 저장 시 Supabase `places` 테이블 UPDATE
- 실시간 반영 (Realtime subscription 기존 활용)

**UX 분기 (클릭 충돌 해결):**
- 행 텍스트 영역 탭 → 수정 모달
- 지도 핀 아이콘 버튼(🗺) → 지도 포커스 (기존 동작 이동)
- 편집 모드 중에는 수정 모달 진입 불가 (드래그 모드 우선)

---

## 2. 상하 스크롤로 일자 넘기기

**현황:** 상단 탭 버튼을 눌러야만 일자 전환 가능.

**요구사항:**
- 장소 리스트 영역(55dvh)에서 스크롤이 맨 위/아래 끝에 도달했을 때 이전/다음 날로 자동 전환
- 전환 애니메이션: 슬라이드 또는 페이드
- 스크롤 끝 감지 조건:
  - `scrollHeight > clientHeight` (실제 스크롤 가능한 경우에만 동작)
  - `scrollTop === 0` + 위 방향 → 이전 Day
  - `scrollTop + clientHeight >= scrollHeight - 1` + 아래 방향 → 다음 Day
- 연속 오발동 방지: 전환 후 300ms 쿨다운
- 장소 수가 적어 스크롤이 없는 경우 오발동 방지: 스크롤 불가 상태에서는 감지 비활성화

---

## 3. 구글 길찾기 바로 연동 버튼

**요구사항:**
- 장소 항목마다 "길찾기" 아이콘 버튼(↗) 추가
- 탭 시 Google Maps 길찾기 URL로 이동:
  `https://www.google.com/maps/dir/?api=1&destination={lat},{lng}&travelmode=transit`
- `destination_place_id` 미사용 (DB에 `google_place_id` 컬럼 없음)
- 현위치 → 해당 장소 경로 검색 (destination만 지정 시 현위치 자동 출발)
- `target="_blank"` + `rel="noopener noreferrer"`
- 편집 모드 중 비활성화

---

## 4. Day 탭에 날짜·요일 표시

**현황:** "Day 1", "Day 2" 텍스트만 표시.

**요구사항:**
- 탭 내 2줄 레이아웃: 윗줄 "Day N" (기존), 아랫줄 "4/15 화" (작은 글씨)
- 날짜 파싱 주의: `day.date`는 `YYYY-MM-DD` UTC 문자열 → `new Date(day.date + 'T00:00:00')` 로컬 파싱 사용 (하루 차이 버그 방지)
- 요일: `['일','월','화','수','목','금','토'][date.getDay()]`
- 선택된 탭은 기존 강조 스타일 유지

---

## 5. Day 삭제 기능

**현황:** Day 추가만 가능. 삭제 없음.

**요구사항:**
- Day 탭 옆 ✕ 버튼으로 삭제
- 해당 Day에 장소가 있을 경우 확인 다이얼로그: "이 날의 장소 N개도 함께 삭제됩니다"
- 삭제 순서 (CASCADE 보장 불가로 코드에서 직접 처리):
  1. `places` 테이블에서 해당 `day_id`의 행 전체 삭제
  2. `days` 테이블에서 해당 `id` 삭제
- 삭제 후 인접 Day로 포커스 이동 (삭제한 Day 인덱스 - 1, 없으면 다음 Day)

---

## 6. 장소 간 거리·소요시간 표시

**요구사항:**
- 장소 리스트에서 인접한 두 장소 사이에 거리/시간 배지 표시
- Google Maps **Routes API** (`computeRouteMatrix`) 사용 — Distance Matrix API는 2025년 3월 Deprecated
- Next.js API Route(`/api/distance`)를 통해 서버에서 호출 (API 키 클라이언트 노출 방지)
- 소요시간 우선순위: **대중교통 > 자동차(택시) > 도보** (가능한 것 순서대로 표시)
- 불가 시 도보 소요시간만 표시
- 거리는 km/m 단위 표시
- 세션 내 캐싱: `Map<string, DistanceResult>` (key: `${placeA.id}:${placeB.id}`)
- API 키에 Routes API 활성화 필요 (유료, 구현 전 콘솔에서 확인)
- API 호출 실패 시 조용히 숨김

## 7. 장소 순서 변경 (편집 모드 토글)

**현황:** `order_key`(fractional-indexing) 필드는 있으나 UI 미제공.

**요구사항:**
- 장소 리스트 헤더에 **편집 모드 토글 버튼** 추가 (연필 아이콘 또는 "편집" 텍스트)
- 편집 모드 ON:
  - 각 장소 왼쪽에 드래그 핸들(⠿) 표시
  - 길게 눌러 드래그로 순서 변경 (`@dnd-kit/sortable` 사용)
  - 길찾기·삭제·수정 버튼 비활성화 (실수 방지)
- 편집 모드 OFF: 드래그 핸들 숨김, 일반 탭/클릭 동작 복원
- 순서 변경 저장: `generateKeyBetween(prevKey, nextKey)` → Supabase `places` UPDATE
